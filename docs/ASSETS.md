# Art & Audio Assets

The app is wired with **per-region art and audio hooks** that no-op gracefully
until you add files — the game is fully playable with zero assets (backgrounds
fall back to region gradients; audio is silent). This doc is the manifest of
every asset the app will use and the exact names it looks for.

## How the hooks work

- Each `RoomID` maps to a `Region` (`Sources/AvasiaEngine/Model/Region.swift`).
- `Media` (`Sources/AvasiaEngine/Content/Media.swift`) derives, per region, a
  background name, a header-illustration name, and an ambient-loop name using a
  fixed convention.
- The app resolves them:
  - **Images** via `UIImage(named:)` from the asset catalog
    (`App/Resources/Assets.xcassets`). Missing → gradient fallback
    (`App/Views/RegionArt.swift`, `RegionPalette`).
  - **Audio** via `AudioManager` (`App/Audio/AudioManager.swift`) from
    `App/Resources/Audio/`. Missing → silence.
- Cues fire from `GameViewModel.submit`: room change → ambient swap + `sfx_move`;
  green item/spell line → `sfx_item`/`sfx_spell`; death → `sfx_death`; win →
  `sfx_win`. Title/credits screens loop `music_title`.

## Image assets (Asset Catalog)

Add each as an **Image Set** in `App/Resources/Assets.xcassets` with the exact
name below (@1x/@2x/@3x as usual). Two per region.

| Region | Background `bg_<region>` | Illustration `art_<region>` |
|---|---|---|
| oceandale | `bg_oceandale` | `art_oceandale` |
| beach | `bg_beach` | `art_beach` |
| graveyard | `bg_graveyard` | `art_graveyard` |
| splitpath | `bg_splitpath` | `art_splitpath` |
| mountain | `bg_mountain` | `art_mountain` |
| cave | `bg_cave` | `art_cave` |
| forest | `bg_forest` | `art_forest` |
| tree | `bg_tree` | `art_tree` |
| road | `bg_road` | `art_road` |
| shore | `bg_shore` | `art_shore` |
| nacastrum | `bg_nacastrum` | `art_nacastrum` |
| aylova | `bg_aylova` | `art_aylova` |
| cataracta | `bg_cataracta` | `art_cataracta` |

- **Background** (`bg_*`): full-bleed, drawn at ~45% opacity behind the text — so
  favor darker, low-contrast images. Portrait-friendly (fills width).
- **Illustration** (`art_*`): the 96pt header band; landscape, fills width and is
  clipped. This is the "scene-setter" for each region.
- Also recommended: `AppIcon` (already stubbed) and a launch image.

Naming is generated by `Media.backgroundName(for:)` / `illustrationName(for:)` —
keep these in sync if you change the convention.

## Audio assets (`App/Resources/Audio/`)

Drop files here; the loader accepts `.m4a`, `.mp3`, `.caf`, or `.wav` (first
match wins). Base names only — no extension in code.

### Ambient loops (one per region) — `amb_<region>`
`amb_oceandale`, `amb_beach`, `amb_graveyard`, `amb_splitpath`, `amb_mountain`,
`amb_cave`, `amb_forest`, `amb_tree`, `amb_road`, `amb_shore`, `amb_nacastrum`,
`amb_aylova`, `amb_cataracta`. Seamless loops; played at 50% volume. Suggested beds: waves
(beach/shore), wind (mountain/road), dripping water (cave), birdsong/leaves
(forest/tree), low hum (nacastrum/aylova).

### Music
- `music_title` — title & credits loop.

### Sound effects (`SoundCue`)
| Cue | File | Fires when |
|---|---|---|
| `.itemGained` | `sfx_item` | a green "You obtained/received…" item line |
| `.spellLearned` | `sfx_spell` | a green line mentioning a spell |
| `.death` | `sfx_death` | the red "You have died." banner |
| `.win` | `sfx_win` | the canonical ending |
| `.move` | `sfx_move` | any room transition (footstep) |

## Adding assets — checklist

1. Drop images into `App/Resources/Assets.xcassets` as image sets with the names
   above (or audio into `App/Resources/Audio/`).
2. `xcodegen generate` (audio folder and catalog are already in the app target's
   `App/` source path).
3. Run — backgrounds/illustrations and ambient/SFX appear automatically; any
   asset still missing keeps its graceful fallback.

## Style direction

Carry the **blue-crystal** accent (STORY.md §8) across regions; the cave leans
pink-crystal, the forest/tree green, the road warm/ominous, Nacastrum & Aylova
cool and regal. `RegionPalette` already encodes these as gradient fallbacks —
match the art to those hues for a consistent look during partial asset coverage.
